# Рабочий процесс “идея → PR → релиз” (для разработки с ИИ)

## 1) Декомпозиция: режь задачу до размера PR
Правило: если не можешь проверить результат за 10–20 минут — задача слишком большая.

Выход: список маленьких PR (как “кирпичики”).

## 2) Спека
Заполни `ai-playbook/03-spec-template.md`.

## 3) Сессия с ИИ (рекомендуемый сценарий)
1) Попроси план + список файлов, которые будут тронуты.
2) Попроси сделать дифф **минимальным**.
3) Попроси добавить тест или хотя бы воспроизводимый сценарий.
4) Попроси указать риски и края.

## 4) Проверки (обязательный “судья”)
Перед мержем всегда:
- линтер/типизация,
- тесты (хотя бы unit),
- быстрый smoke test вручную (1–2 сценария).

Если проверок нет — твой следующий PR должен быть “добавить проверки”.

## 5) Ревью
Используй `ai-playbook/05-pr-checklist.md`.
Если ты не уверен — попроси “агента‑критика” сделать ревью только текстом.

## 6) Лог изменений
Каждый PR заканчивается коротким описанием:
- что изменилось,
- как проверить,
- какие риски,
- как откатиться.

## 7) Не ускоряй там, где цена ошибки высокая
Auth/security/данные/миграции делай отдельными PR с повышенной строгостью:
больше тестов, меньше изменений, больше проверок.
