# Чек-лист ревью PR от ИИ (production-ready)

## Поведение
- Изменение соответствует спеки/acceptance criteria.
- Нет скрытого изменения поведения “в соседних местах”.
- Нет “магии”: всё явно, названия читаемые, ошибки обработаны.

## Инварианты и края
- Проверены edge cases из спеки.
- Понятно, что будет при недоступности внешних сервисов.
- Нет silent-fail (ошибка не проглатывается без логов).

## Качество и поддерживаемость
- Изменения небольшие и локальные (малый diff).
- Нет лишних рефакторингов “заодно”.
- Нет дублирования логики (или есть явное объяснение).

## Безопасность
- Нет секретов в коде/логах.
- Авторизация/роли не ослаблены “для теста”.
- Валидация входных данных есть на границе.

## Тесты и проверка
- Есть тесты на новое поведение (или объяснение, почему нельзя).
- Локально запущены линтер/тесты, результат известен.

## Наблюдаемость
- Ключевые события/ошибки логируются.
- Логи не содержат PII/секретов.
