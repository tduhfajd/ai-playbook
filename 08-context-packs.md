# Context packs: как давать ИИ “правильный” контекст

## Идея
Не давай “весь код”. Давай контуры системы и контракты, чтобы ИИ не угадывал.

## Pack A — “API change”
Дай:
- route файл,
- контроллер/handler,
- DTO/schema/валидатор,
- пару примеров request/response,
- список edge cases.

## Pack B — “DB change”
Дай:
- текущую схему таблицы (DDL),
- список запросов, которые её используют,
- миграции (если есть),
- требования к обратной совместимости.

## Pack C — “Async / workers”
Дай:
- producer (где ставится задача),
- consumer/worker task,
- формат сообщения/очередь,
- ожидания по retries/timeouts/idempotency.

## Pack D — “UI change”
Дай:
- компонент/страницу,
- API вызовы,
- состояние/сторы,
- UX критерии.

## Практическое правило
Всегда добавляй “что нельзя трогать” и “что не должно измениться”.
