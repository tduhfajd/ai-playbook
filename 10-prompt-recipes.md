# Готовые промпты (копируй-вставляй)

Ниже — набор “боевых” промптов под разработку с ИИ. Подставляй свои детали и ссылки на файлы.

## 1) Промпт: “Сделай план и спроси вопросы”
```
Ты — инженерный агент в репозитории. Работай строго по правилам из ai-playbook/02-agent-policy.md.

Задача: <кратко>

Контекст:
- Файлы: <список путей>
- Текущее поведение: <1-3 пункта>
- Целевое поведение: <1-3 пункта>

Сначала:
1) задай до 7 уточняющих вопросов (только критические),
2) предложи план из 3–7 шагов,
3) перечисли, какие файлы/модули ты планируешь менять и почему.
К коду не переходи, пока не подтвердят план.
```

## 2) Промпт: “Сделай маленький PR (дифф + тест + команды проверки)”
```
Работай строго по ai-playbook/02-agent-policy.md. Никаких массовых изменений.

Сделай реализацию задачи по спеки:
<вставь заполненный шаблон из ai-playbook/03-spec-template.md>

Требования к результату:
1) минимальный diff,
2) добавь/обнови тест(ы), который падает на старом коде и проходит на новом,
3) дай команды для проверки (lint/test/build),
4) перечисли риски/края (что могло сломаться).
```

## 3) Промпт: “Агент‑критик (ревью без правок)”
```
Ты — агент‑критик. Тебе запрещено предлагать изменения кода и запрещено писать патчи.
Твоя задача — сделать ревью по чек‑листу ai-playbook/05-pr-checklist.md и найти риски/регрессии.

Дано:
- Тема PR: <...>
- Список файлов изменён: <...>
- Описание изменений: <...>

Ответь структурировано:
1) Что поменялось в поведении
2) Какие инварианты могли быть нарушены
3) Edge cases и деградация
4) Security риски
5) Какие проверки/тесты обязательно запустить
6) Вердикт: approve / request changes (и почему)
```

## 4) Промпт: “Найди причину падения тестов и предложи минимальный фикс”
```
Работай по ai-playbook/02-agent-policy.md. Никаких переписываний.

Вот вывод ошибки/падающего теста:
<вставь лог>

Сделай:
1) объясни вероятную причину (1–3 гипотезы),
2) выбери самую вероятную,
3) предложи минимальный патч,
4) скажи, какие тесты/команды прогнать после фикса.
```

## 5) Промпт: “Подготовь Context pack”
```
Я хочу изменить <API/DB/async/UI>. Составь список конкретных файлов, которые нужно показать ИИ,
чтобы он не угадывал (по ai-playbook/08-context-packs.md).

Ограничения: не более 10 файлов.
```

## 6) Промпт: “Сделай ADR”
```
Сделай ADR по шаблону ai-playbook/09-adr-template.md.

Решение: <описание>
Контекст: <описание проблемы>
Альтернативы: <если есть>
План PR-ов: <примерный>
```
