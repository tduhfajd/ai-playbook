# AI Playbook — как довести проект до production с помощью ИИ

Этот каталог — практические инструкции “как работать с ИИ, чтобы он ускорял, а не ломал”.
Разложено по пошаговому процессу, шаблонам и чек‑листам.

## Быстрый старт (если ты не пишешь код сам)
1) Начинай с маленькой задачи (≤ 1–3 файла, ≤ 200 строк изменений).
2) Пиши задачу по шаблону: [`03-spec-template.md`](./03-spec-template.md).
3) Включай рельсы для агента: [`02-agent-policy.md`](./02-agent-policy.md).
4) Проси ИИ сделать план → дифф → тесты → короткое “что сломать мог”.
5) Мерж только когда автопроверки зелёные и пройден чек‑лист: [`05-pr-checklist.md`](./05-pr-checklist.md).

## Навигация
- [`01-principles.md`](./01-principles.md) — почему ИИ бустит/ломает и что делать.
- [`02-agent-policy.md`](./02-agent-policy.md) — правила и ограничения для ИИ (guardrails).
- [`03-spec-template.md`](./03-spec-template.md) — шаблон постановки задачи (спека).
- [`04-workflow.md`](./04-workflow.md) — пошаговый цикл “идея → PR → релиз”.
- [`05-pr-checklist.md`](./05-pr-checklist.md) — чек‑лист ревью PR от ИИ.
- [`06-testing.md`](./06-testing.md) — минимальная стратегия тестов + что делать, если тестов нет.
- [`07-debugging.md`](./07-debugging.md) — как чинить баги вместе с ИИ без хаоса.
- [`08-context-packs.md`](./08-context-packs.md) — как давать ИИ “правильный” контекст (контракты, не весь код).
- [`09-adr-template.md`](./09-adr-template.md) — шаблон архитектурного решения (ADR).
- [`10-prompt-recipes.md`](./10-prompt-recipes.md) — готовые промпты (автор/критик/спека/дебаг).
- [`11-definition-of-done.md`](./11-definition-of-done.md) — Definition of Done для задач и PR.
- [`12-agent-artifacts.md`](./12-agent-artifacts.md) — артефакты агента: AGENTS/PLANS/handoff/acceptance.

## Быстрый чек‑лист старта
1) Прочитай `AGENTS.md`.
2) Обнови `PLANS.md` под задачу.
3) Заполни `evals/acceptance.md`.
4) Задай агенту: “покажи план и список файлов до кода”.
5) В конце сессии обнови `handoff.md`.
